## 기존 게임 규칙

- 총 20문제, 문제당 5초
- 1~18번은 4지선다, 19~20번 보스 문제는 8지선다
- 16번부터 같은 일의 자리를 유도하는 10단위 함정 보기 사용
- 바로 직전의 곱셈식 한 문제만 중복 방지하며 최대 10회 재추첨
- 기본 점수는 정답당 10점
- 속도 점수는 `floor(5 - 응답 시간)`이며 최소 0점
- 로그인 사용자는 기존 플레이 횟수 × 2점을 추가로 획득
- 누적 150점마다 레벨 1 증가
- 레벨이 오를 때 정답률별 확률로 N/R/SR/UR 고양이 지급

## 데이터 저장 방식

수정 전에는 `localStorage`, `sessionStorage`, IndexedDB를 사용하지 않았습니다. 계정, 비밀번호, 누적 점수, 레벨, 플레이 횟수, 보상 배열은 Firestore `users/{username}` 문서에 저장됩니다. 신규 V2 기초 저장소는 별도 키 `gugudanV2Save`를 사용하므로 기존 Firebase 데이터와 충돌하지 않습니다.

## Firebase 및 랭킹

- Firebase 프로젝트: `gugu-cat-adventrue`
- 컬렉션: `users`
- 전체 랭킹: `totalPoints` 내림차순, 상위 5명
- 주간 랭킹: 기존 구조에 없음
- Firebase 설정값, 컬렉션명, 읽기·쓰기 형식은 변경하지 않았습니다.

## 고양이와 이미지

기존 보상은 `{ id, rank, bg, border, title }` 구조이며 등급은 N/R/SR/UR입니다. 이미지는 파일이 아니라 `robohash.org/{id}` URL에서 생성합니다. 기존 획득 배열은 삭제하거나 변환하지 않고 `legacyRewards`로 보존할 수 있는 마이그레이션 기반을 추가했습니다.

## 사운드

기존에는 Web Audio 오실레이터로 정답, 오답, 전설, 보스 사이렌을 생성합니다. 음원 파일은 없습니다. V2 사운드 목록과 서비스는 별도로 추가했으며 파일이 없을 때 조용히 건너뜁니다. 기존 합성 효과음은 유지했습니다.

## 모바일 및 접근성

기존 최대 너비는 600px이고 400px 이하 미디어 쿼리가 있습니다. 답 버튼은 터치하기 충분한 크기입니다. 일부 이미지에 `alt`가 없었고 빠른 연속 답변을 막는 명시적 잠금 변수가 없었습니다.

## 발견한 문제점

- 전역 변수와 UI·게임·데이터·Firebase 코드가 한 파일에 강하게 결합됨
- Firebase SDK가 로드되지 않으면 초기화 단계에서 게스트 게임까지 중단될 수 있음
- 관리자 계정과 사용자 비밀번호가 클라이언트 코드/Firestore 평문 구조에 의존함
- 기존 관리자 화면은 콘텐츠 운영용이 아니라 사용자 계정 관리 기능임
- 고양이 데이터가 확정 카탈로그가 아닌 매 보상 시 무작위 ID로 생성됨
- 네트워크 이미지 실패 처리와 로컬 저장 버전이 없음
- 실제 브라우저 통합 테스트 환경과 배포 자동 검증이 없음

## 향후 확장이 어려운 부분

문제 난이도 분기, 점수, 보상 확률, DOM 변경이 `app.js` 흐름에 여전히 연결되어 있습니다. 1단계에서는 회귀 위험을 줄이기 위해 핵심 계산을 그대로 두고 설정·상태·데이터·서비스 인터페이스부터 분리했습니다. 다음 단계에서 UI 흐름을 서비스로 점진 이동해야 합니다.
