# 냥코 구구단 대모험 V2 — 2단계 분석

분석 기준: 1단계 완료 상태, 2026-07-21

## 재사용 가능한 기존 기능

- `js/config/gameConfig.js`: 클래식 규칙과 비활성 모드 설정
- `js/game/gameState.js`: 공통 상태와 정오답 기록
- `js/game/questionGenerator.js`: 중복 없는 오답 보기 및 오름차순 정렬
- `js/game/scoreManager.js`: 기존 클래식 점수 계산
- `js/services/storageService.js`: 오류 안전 로컬 저장과 스키마 마이그레이션
- `js/services/soundService.js`: 파일 누락과 재생 거부를 허용하는 사운드 서비스
- `js/services/rankingService.js`: 1단계 인터페이스
- `js/app.js`: 화면 전환, Firebase 초기화, 기존 클래식, 도감, `users` 전체 랭킹
- `play-screen`: 모든 모드가 공유할 문제·보기·피드백 화면

## 수정이 필요한 기능

기존 플레이 흐름은 20문제 클래식에 고정되어 있습니다. 시간 제한형 종료, 목숨형 종료, 공통 결과, 모드별 기록, 모드별 랭킹을 별도 규칙 엔진으로 연결해야 합니다. 저장 스키마는 2에서 3으로 확장하고 Firebase 랭킹 서비스는 실제 구현이 필요합니다.

## 모드별 분리

- 클래식: 기존 `app.js` 흐름을 그대로 유지
- 타임어택: 실제 종료 예정 시각, 60초, 콤보 점수, 즉시 다음 문제
- 무제한: 3목숨, 오답 시 1 감소, 10정답마다 난이도 상승
- 어드벤처: 설정과 비활성 버튼만 유지

화면은 복사하지 않고 기존 `play-screen`과 `result-screen`을 재사용합니다.

## Firebase 랭킹 구조

기존 클래식은 `users` 문서의 `totalPoints`를 기준으로 상위 5명을 조회합니다. 이 구조는 모드별 세션 기록에 적합하지 않으므로 변경하지 않습니다. 신규 모드는 별도 `modeRankings/{sessionId}` 컬렉션을 사용합니다.

## 주간 랭킹 방식

한국 시간(UTC+09:00) 월요일 00:00부터 다음 월요일 00:00 미만의 반개방 구간을 UTC `Date`로 계산합니다. 저장 시 Firebase 서버 타임스탬프를 사용하고 주간 조회는 `playedAtTimestamp` 범위 쿼리 후 클라이언트에서 모드별 기준으로 정렬·사용자별 최고 1건을 표시합니다.

## 기존 데이터 호환

스키마 2의 닉네임, 선택 고양이, 점수, 최고 콤보, 획득 기록, 설정을 스키마 3에 병합합니다. 기존 Firebase `users` 문서와 고양이 보상 배열은 수정하지 않습니다. 신규 익명 `playerId`만 로컬에서 생성합니다.

## 위험 요소

- Firebase의 실제 보안 규칙과 복합 인덱스는 로컬 코드에서 확인할 수 없음
- 클라이언트 검증만으로 점수 조작을 완전히 방지할 수 없음
- 서버 타임스탬프는 저장 후 확정되므로 오프라인 제출은 온라인 랭킹에 반영되지 않음
- 기존 클라이언트 비밀번호·관리자 구조는 보안상 개선이 필요하지만 이번 범위에서는 유지
- 브라우저 백그라운드 제한은 발생할 수 있으나 타임어택은 interval 횟수가 아닌 `Date.now()`로 보정
