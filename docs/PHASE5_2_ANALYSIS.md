# Phase 5.2 사전 분석

- 프로젝트는 Firebase Authentication이 아니라 Firestore `users/{username}` 문서와 비밀번호 비교를 사용한다. 따라서 현재 사용 가능한 안정 식별자는 인증된 사용자 문서 ID다.
- 기존 로컬 저장소는 `gugudanV2Save` 한 개를 공유하여 브라우저에서 계정을 바꾸면 수집·재화·모험 데이터가 섞일 수 있었다.
- 도감 그리드는 보유 카드만 렌더링하지만 카드 폭이 커서 모바일 탐색성이 낮았다.
- 시즌 미션은 존재하지만 한국 날짜로 매일 초기화되는 독립 일일 미션은 없었다.
- 랭킹 컬렉션은 실제 코드상 `modeRankings`이며 현재 온라인 제출 모드는 타임어택이다. 기존 주간 필드는 호환을 위해 유지한다.

위험을 줄이기 위해 저장 API 호출부는 유지하고 그 아래에 계정 컨텍스트 계층을 추가했다. 공용 저장 데이터는 최초 인증 계정 한 곳에만 이전하며 다른 계정에는 복제하지 않는다.
