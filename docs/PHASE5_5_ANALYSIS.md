# Phase 5.5 분석

- 보스 대표 고양이 오류: `player-cat-image`와 `player-cat-name` DOM은 있었지만 모험 시작 코드가 값을 설정하지 않았다. 스킬 HUD 이미지만 설정되고 보스 카드 이미지는 빈 상태였다.
- fallback: `releasePolicyService.getSelectedCat()`이 보유·공개 상태를 검사하고 기본 고양이로 복구하며, `assetLoader`가 이미지 실패를 대체한다.
- 보스 UI: 월드별 색상 구분과 대표 고양이 정보가 없고 작은 절대 위치 박스만 사용했다.
- 월드맵: 데스크톱 CSS의 `.world-card:first-child`가 첫 카드인 월드 1을 항상 전체 폭으로 만들었다.
- 도감: 별 영역 추가 뒤 112px 폭, 154px 최소 높이에 여러 후행 이미지 크기 규칙이 겹쳐 이미지 공간이 눌렸다.
- 뽑기: 문이 1.35초에 열리지만 결과 전환 시 집이 남아 카드와 겹쳤다. 결과 설명도 렌더링하지 않았다.
- 설명: 상세 컨트롤러에는 연결됐지만 뽑기 결과와 보스 동행 카드에는 연결되지 않았다.
- 효과: effectThemeId는 레이어 속성까지만 연결됐고 실제 테마 CSS가 부족했다. soundThemeId는 재생 함수가 사용하지 않았다.
- 수정: 스토리 데이터/서비스/모험 엔진, 대표 고양이 사운드 런타임, Phase 5.4 뽑기 결과, CSS, index와 테스트를 변경했다.
