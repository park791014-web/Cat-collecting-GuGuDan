# 냥코 구구단 대모험 V2 — 2단계 완료 보고서

## 1. 작업 개요

기존 클래식과 도감을 유지하면서 공통 문제 화면에 타임어택과 무제한 도전을 추가했습니다. 스키마 3 로컬 기록, 개인 최고, 모드별 주간·전체 Firebase 랭킹, 공통 결과·순위 UI를 구현했습니다.

## 2. 수정한 파일

- `index.html`: 모드 선택, 상태 표시, 공통 결과, 랭킹 탭, 음소거 UI
- `css/style.css`: 모드 카드, 모바일 상태·결과·랭킹 스타일
- `js/config/gameConfig.js`: Phase 2 버전과 활성 모드 규칙
- `js/game/gameState.js`: 세션·시간·목숨·난이도·입력 잠금·종료 이유
- `js/game/scoreManager.js`: 콤보 보너스
- `js/services/storageService.js`: 스키마 3, 최고 기록, 최근 기록
- `js/services/rankingService.js`: 실제 Firebase 제출·조회·한국 주간 범위
- `js/app.js`: Firebase 서비스 연결과 음소거, 모드 UI 호환

## 4. 타임어택

60초 종료 예정 시각을 `Date.now()`로 계산하고 100ms마다 표시만 갱신합니다. 정답은 즉시 다음 문제로 진행하며 오답은 기본 점수·시간 차감이 없습니다. 마지막 10초에는 숫자·배경·타이머 색상을 함께 강조하며 빠른 점멸은 사용하지 않습니다.

## 5. 무제한 도전

3목숨으로 시작해 오답마다 1을 차감하고 0에서 한 번만 종료합니다. 정답 10개마다 난이도가 상승합니다. 1단계는 2~5단·4보기, 2단계는 2~9단·4보기, 3단계 이상은 2~9단·6보기와 유사 오답을 사용합니다. 빈칸 문제는 추가하지 않았습니다.

## 6. 타이머 정확성

interval 호출 횟수에 의존하지 않고 `endAt - Date.now()`로 남은 시간을 계산합니다. 탭이 비활성화되었다가 돌아와도 실제 종료 시각을 기준으로 0초 처리합니다. 재시작·종료·화면 이탈에서 timer와 전환 timeout을 모두 정리합니다.

## 7. 난이도 상승

무제한 모드의 누적 정답 수가 10의 배수가 될 때 `difficultyLevel`을 1 올립니다. 범위, 역순 표시 확률, 보기 수, 최근 문제 기억, 함정 보기를 단계별로 강화합니다.

## 8. 점수와 콤보

모드 정답 기본 점수는 10점입니다. 새 콤보 기준 1~4는 +0, 5~9는 +2, 10~19는 +5, 20 이상은 +8입니다. 오답은 점수를 깎지 않고 콤보를 0으로 초기화합니다. 기존 클래식 계산은 변경하지 않았습니다.

## 9. 개인 최고 기록

타임어택 비교 순서는 점수, 정답 수, 최고 콤보, 적은 오답입니다. 무제한은 정답 수, 점수, 도달 난이도, 최고 콤보 순입니다. 낮은 기록으로 덮어쓰지 않으며 달성 시각을 저장하고 결과에서 강조합니다.

## 10. 게임 기록

민감 정보 없이 세션 ID, 모드, 점수, 정오답, 정확도, 최고 콤보, 난이도, 제한 시간, 종료 이유, 시각을 로컬에 저장합니다. 손상 항목은 제외하며 최신 50건만 유지합니다. Firebase 실패와 무관하게 먼저 저장합니다.

## 11. 주간 순위

한국 시간 월요일 00:00 이상, 다음 월요일 00:00 미만입니다. `getKoreanWeekRange()`가 UTC `Date` 범위와 한국 날짜 라벨을 반환합니다.

## 12. 한국 시간 처리

기준 시각에 9시간을 더한 뒤 UTC 달력 연산으로 한국 요일과 날짜를 구하고, 월요일 00시를 다시 UTC로 변환합니다. 브라우저가 어느 시간대에 있어도 같은 범위를 계산합니다.

## 13. Firebase 컬렉션

기존 클래식: `users/{username}` 유지. 신규 모드: `modeRankings/{sessionId}`. 신규 문서는 닉네임, 익명 playerId, 모드, 점수, 정오답, 정확도, 콤보, 난이도, 제한 시간, ISO 플레이 시각, 서버 타임스탬프, 앱 버전을 포함합니다.

## 14. 중복 정책

세션 ID를 문서 ID로 사용하고 현재 페이지에서 동일 세션의 재제출도 차단합니다. 모든 세션은 저장하지만 화면에서는 playerId, rankingId, 닉네임 순으로 사용자 키를 선택하여 최고 기록 1건만 표시합니다.

## 15. 기존 랭킹 호환

클래식 `users` 상위 5명 조회를 그대로 유지했습니다. 신규 컬렉션을 사용하므로 기존 문서·보안 규칙·도감 데이터와 필드 충돌이 없습니다.

## 17. 발견된 문제

Firebase 실제 규칙과 인덱스는 저장소에 없어 자동 확인할 수 없습니다. 실시간 네트워크 제출·조회는 배포 환경에서 확인해야 합니다. 클라이언트 검증만으로 치팅을 완전히 방지할 수 없습니다.

## 18. 수동 확인

Firebase 인증 사용자로 신규 모드 기록 제출, 두 복합 인덱스, 보안 규칙 허용 필드, 모바일 실제 기기 320~768px, 외부 음원 추가 후 음소거와 중첩 재생을 확인하십시오.

## 사용자 요청 후 조정

기본 도전과 차이가 충분히 드러나지 않는 무제한 도전은 설정에서 비활성화하고 모드 선택 및 순위 탭에서 숨겼습니다. 순위 화면의 기본 선택은 기존 데이터와 호환되는 `기본 도전 / 전체`로 변경했습니다.

이후 Firebase 인덱스 진단 요청에 따라 신규 기록에 한국 시간 월요일 기준 `weekKey`를 추가했습니다. 전체 순위는 `mode + score`, 주간 순위는 `mode + weekKey + score` 쿼리로 통일했습니다. 쿼리 오류 전체와 Firebase가 제공한 인덱스 생성 URL을 개발자 콘솔에 출력합니다. `weekKey`가 없는 기존 문서는 주간 순위에서 제외됩니다.
