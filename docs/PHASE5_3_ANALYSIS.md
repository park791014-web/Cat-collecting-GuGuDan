# Phase 5.3 사전 분석

## 확인 결과

- 시즌 활성화가 여러 렌더러에 흩어져 있어 `js/config/featureFlags.js`로 통합했다. 로비 배너 호출과 시즌 미션 기록을 플래그로 차단하고 시즌 화면 DOM은 배포 HTML에서 생성하지 않는다.
- 시즌 원본 데이터, 서비스, 이미지, 티켓 및 진행 저장값은 삭제하지 않는다. 시즌 대표 고양이는 `hiddenSeasonSelectedCatId`에 보존하고 보유한 첫 기본 고양이로 임시 교체한다.
- 기본 게임의 직접 고양이 지급은 `js/app.js`의 레벨 상승 블록에 있었다. 해당 지급과 Robohash 결과 UI를 제거했다.
- 기존 기본 게임은 V2 코인 보상이 없었고 Firebase 점수만 저장했다. 신규 보상 서비스가 세션 ID별 한 번만 코인을 지급한다.
- 티켓 카드팩은 티켓 차감과 카드 반영 후 한 번 저장했다. 같은 원자적 패턴을 코인 뽑기에 적용하고 후보는 기본 컬렉션으로 제한했다.
- 물음표 이펙트 원인은 `css/style.css`의 손상된 `content:"??"`, `content:"?"`였다. 문자 파티클을 비활성화하고 로컬 SVG 레이어로 교체했다.
- 스토리 버튼은 `.option-btn`, 기본/타임어택은 `.btn-answer`라 Phase 5.2 크기 규칙을 공유하지 못했다. 두 선택자를 공통 변수로 묶었다.
- 스키마 마이그레이션은 필요 없다. 선택 복구 필드와 보상 이력 배열은 기존 병합 저장 구조에서 선택적으로 보존된다.

## 수정 범위

설정, 출시 정책, 클래식 보상, 코인 뽑기, 이펙트 서비스, 도감/뽑기 UI, 기본 게임, 모험 엔진, 공통 CSS, 로컬 SVG와 문서를 수정했다. Firebase 설정과 보안 규칙은 변경하지 않았다.
