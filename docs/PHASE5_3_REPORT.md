# Phase 5.3 완료 보고서

## 시즌 비공개와 데이터 보존

중앙 기능 플래그 4개를 기본 `false`로 설정했다. 배포 HTML에서 시즌 화면을 생성하지 않고 로비 배너 및 시즌 미션 기록도 차단했다. 시즌 코드·데이터·이미지·티켓·진행·보유 정보는 그대로 보존한다. 선택된 시즌 고양이는 `hiddenSeasonSelectedCatId`에 보관하고 기본 고양이를 임시 장착하며, 테스트 플래그 재활성화 시 복구한다.

## 보상과 뽑기

레벨 상승 시 직접 생성하던 고양이 및 원격 Robohash 표시를 제거했다. 기본 게임은 완료 30 + 정답당 5 + 완벽 30 + 개인기록 50코인을 지급한다. `classicRewardedSessionIds`로 중복 지급을 막고 결과 화면에 항목별 내역을 표시한다. 뽑기는 500코인, 확률 65/27/7/1이며 기본 고양이만 후보로 삼는다. 한 번 저장, 입력 잠금, 잔액 재검사와 음수 방지를 적용했다. 티켓 값은 보존하되 UI에서 숨겼다.

## 이펙트와 스토리 버튼

손상된 CSS 문자 파티클이 `?`를 출력한 원인이었다. 5개 로컬 SVG, 고정 공통 레이어, 화면 폭별 무작위 위치·크기·회전·시간, 콤보 강화, 20개 상한과 자동 정리를 적용했다. 이미지 오류는 요소 제거로 처리한다. 모션 감소도 지원한다. `.option-btn`과 `.btn-answer`를 최소 64px, 24~38px 글자, 10px 간격으로 통일하고 모바일은 2열, 넓은 화면은 4열로 구성했다. 도감은 `cat-thumbnail-item` 범위 클래스를 사용한다.

## 검증과 남은 수동 작업

정적 문법 검사, 설정 확률/비용, 시즌 DOM 부재, SVG 존재, 문자 파티클 비활성 및 기존 자동 테스트를 실행한다. Firebase 설정과 규칙은 수정하지 않았다. 실제 터치 기기 320/360/390/412/768px의 시각·성능과 로그인 Firebase 흐름은 브라우저/계정 환경에서 최종 수동 확인이 필요하다.
