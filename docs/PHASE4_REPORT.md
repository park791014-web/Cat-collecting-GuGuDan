# Phase 4 완료 보고서

확인 날짜: 2026-07-22

## 1. 작업 개요와 분석 결과

기존 카드 UI, 고양이 20종, 모험 진행과 Firebase 순위 구조를 유지하면서 보상·카드팩·수집 진행·장착·공통 스킬 엔진을 추가했습니다. 세부 사전 분석은 `PHASE4_ANALYSIS.md`에 기록했습니다.

## 2. 생성한 파일

- 설정: `rewardConfig.js`, `cardPackConfig.js`, `skillConfig.js`
- 서비스: `rewardService.js`, `cardPackService.js`, `skillEngine.js`
- UI: `collectionController.js`
- 검증: `phase4Validators.js`, `phase4Validation.test.js`
- 문서: Phase 4 분석, 보고서, 밸런스 설정

## 3. 수정한 파일

`index.html`, `css/style.css`, `js/app.js`, `js/data/cats.js`, `js/game/adventureEngine.js`, `js/game/modeEngine.js`, `js/services/storageService.js`, `js/services/soundService.js`와 기존 검증 테스트를 수정했습니다.

## 4. 재화와 스테이지 보상

코인, 일반 티켓, 고급 티켓을 스키마 5에 저장합니다. 일반·중간 보스·최종 보스의 최초 보상, 일반 반복 보상, 별 1~3의 누적 차액을 설정 기반으로 계산합니다. 세션 ID, 최초 클리어 목록, 스테이지별 지급 별 기록으로 중복 지급을 막고 최근 세션 100개만 유지합니다.

## 5. 카드팩과 수집

일반·고급 팩의 확률 합을 검증하고 누적 확률로 등급을 선택한 뒤 해당 기본 등급에서 균등 추첨합니다. legacy와 시즌 카드는 제외합니다. 티켓 확인부터 카드·중복 결과 반영까지 메모리에서 계산한 뒤 한 번 저장합니다. 신규 카드는 획득일과 진행 데이터를 만들고, 중복 카드는 고양이별 중복 횟수와 등급 조각을 증가시킵니다.

## 6. 대표 고양이와 상세 화면

기존 도감 카드를 누르면 획득일, 중복 횟수, 스킬과 적용 모드가 포함된 포커스 가능한 상세 모달이 열립니다. 보유한 기본 고양이만 장착할 수 있고 `profile.selectedCatId`에 저장됩니다. 메인, 준비 화면, 플레이 HUD, 보스전과 상세 화면에 표시합니다.

## 7. 영웅·전설 스킬

- 번개의 야옹: 10의 배수 콤보별 한 번만 시각·소리 연출
- 냥냥 실드: 모험 첫 오답의 목숨 감소 1회 방어
- 지혜의 발톱: 기본·모험에서 보기 4개 이상일 때 오답 하나 1회 제거
- 시간의 발바닥: 모험 시간 스테이지 시작 시 2초 추가
- 황금의 포효: 모험 보스 입장 시 황금 연출
- 아홉 번째 기회: 모험 목숨 0에서 한 번 목숨 1로 복구

스킬 효과음 파일이 없어도 Web Audio의 짧은 합성 효과음으로 처리하며, 소리 설정이 꺼져도 시각 안내는 유지합니다.

## 8. 타임어택 공정성

공식 타임어택은 순수 연출형 번개만 허용합니다. 시간·보기·목숨·부활·점수 효과는 차단합니다. 결과에 `rankedSkillPolicy: "cosmetic_only"`를 기록하지만 기존 점수 계산, Firebase 컬렉션, 주간·전체 쿼리는 변경하지 않았습니다.

## 9. 저장과 마이그레이션

스키마는 5입니다. 스키마 1~4의 닉네임, playerId, 대표·보유 고양이, 기본·타임어택 기록, 모험 해제와 별, 음량, legacy 무제한 기록을 유지합니다. 새 재화·조각·고양이 진행·보상 이력만 추가하며 손상된 일부 숫자는 안전한 기본값으로 복구합니다.

## 10. 모바일·접근성

320px에서 팩 결과와 모달을 축소하고 스크롤 가능하게 했습니다. 티켓 부족은 텍스트와 비활성 버튼으로 표시합니다. 팩 확률은 텍스트로 제공하고, 개봉은 탭으로 건너뛸 수 있습니다. 상세 모달 포커스 복귀, 명확한 장착 레이블, 스킬 상태 live 안내, 모션 감소 설정을 적용했습니다.

## 11. 테스트와 발견 사항

자동 테스트로 스키마 이전, 설정 검증, 최초·반복·별 보상, 같은 세션 중복 차단, 티켓 차감, 신규·중복 카드, 조각, 번개 중복 방지, 실드·부활 1회, 타임어택 성능 스킬 차단을 확인했습니다. 기존 Phase 1~3 검증과 구문·상대경로 검사도 함께 실행했습니다.

로컬 `file://` 탭이 자동 브라우저 제어 세션에 노출되지 않아 실제 모바일 픽셀 클릭 검증과 Firebase 실서비스 제출은 수동 확인 사항입니다. Firebase 설정과 규칙은 변경하지 않았습니다.

## 12. Phase 5 준비 상태

확률·보상·스킬 정책이 설정 파일에 분리되어 관리자 조정, 조각 기반 꾸미기, 고품질 카드 이미지 교체를 확장할 수 있습니다. 시즌 카드와 유료 기능은 포함하지 않았습니다.
