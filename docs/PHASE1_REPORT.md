# 냥코 구구단 대모험 V2 — 1단계 완료 보고서

## 1. 작업 개요

기존 단일 HTML 정적 게임을 백업하고, GitHub Pages 배포 호환성을 유지한 채 CSS와 JavaScript를 점진 분리했습니다. 기존 플레이 규칙을 유지하면서 V2 확장용 설정, 데이터, 상태, 서비스, 검증 기반을 추가했습니다.

## 2. 분석한 기존 구조

세부 분석은 `PHASE1_ANALYSIS.md`에 기록했습니다. 기존 게임은 Firestore `users` 컬렉션과 원격 Robohash 이미지를 사용하며 브라우저 로컬 저장소는 사용하지 않았습니다.

## 3. 생성한 파일

- `css/style.css`
- `js/app.js`
- `js/config/`: 게임·고양이·시즌·사운드 설정
- `js/data/`: 고양이 20종·시즌·향후 스테이지 데이터
- `js/game/`: 상태·문제 보기·점수 모듈
- `js/services/`: 저장·사운드·랭킹 인터페이스
- `js/utils/`: 이미지 실패 처리·데이터 검증
- `js/tests/dataValidation.test.js`
- `backup/phase1-original/`
- `docs/PHASE1_ANALYSIS.md`, `docs/PHASE1_REPORT.md`, `docs/ASSET_REQUIREMENTS.md`
- 향후 에셋용 빈 디렉터리 구조

## 4. 수정한 파일

`index.html`에서 인라인 CSS/JavaScript를 외부 정적 파일로 분리하고 V2 버전 표시와 이미지 설명을 추가했습니다. `js/app.js`에는 중앙 설정 연결, 상태 기록, Firebase 초기화 보호, 연속 입력 잠금, 이미지 실패 처리를 연결했습니다.

## 5. 백업한 파일

수정 전 `index.html`을 `backup/phase1-original/index.html`로 복사했습니다. 원본과 백업 SHA-256은 `0022AD402C3AAFCCAC38ECD38FCA221D5628D54A4A7B3693EDCEC0CFBCB930F0`으로 일치합니다.

## 6. 고양이 20마리 데이터 구성

- 일반 8: 하얀, 까만, 치즈, 회색, 삼색, 얼룩, 턱시도, 갈색 냥이
- 희귀 6: 별무늬, 달빛, 구름, 무지개, 보석, 왕관 냥이
- 영웅 4: 번개, 방패, 지혜, 시간 영웅 냥이
- 전설 2: 황금 전설, 아홉 목숨 전설 냥이

기존 N/R/SR/UR 보상은 삭제하지 않고 기존 도감에서 계속 표시합니다.

## 7. 스킬 데이터 구성

일반·희귀에는 스킬이 없고 영웅·전설에는 각각 하나가 있습니다. 10콤보 소리, 첫 오답 방어, 오답 보기 제거, 시간 추가, 보스 효과, 1회 부활 데이터 형식을 마련했으며 실행 기능은 아직 활성화하지 않았습니다.

## 8. 시즌 데이터 구조

20종을 포함한 항상 활성 상태의 `base` 영구 시즌을 추가했습니다. 종료 정책은 `permanent`, `unavailable`, `limited_return`, `special_shop`을 검증합니다.

## 9. 사운드 데이터 구조

정답 3개와 오답 2개의 예정 경로를 등록했습니다. 파일이 없으므로 기본값은 비활성입니다. 첫 포인터 입력 이후 초기화, Promise 오류 흡수, 볼륨·고양이 음성 설정 저장, 전체 정지를 지원합니다. 기존 Web Audio 합성음은 유지했습니다.

## 10. 저장 데이터 마이그레이션

`gugudanV2Save` 스키마 버전 2를 추가했습니다. 이전 형태의 닉네임, 점수, 최고 콤보, 사용자/랭킹 ID, 획득 고양이를 보존합니다. 손상 JSON과 저장소 접근 실패는 기본값으로 복구하며 자동 초기화하지 않습니다. 기존 Firebase 문서 구조는 변경하지 않았습니다.

## 11. Firebase 유지 사항

설정값, 프로젝트 ID, `users` 컬렉션, 점수 저장, 상위 5명 전체 랭킹을 유지했습니다. SDK 초기화 실패 시 경고만 남기고 게스트 게임 로딩을 계속합니다. 주간 랭킹은 비활성 서비스 응답만 제공합니다.

## 12. 테스트 결과

- 모든 JavaScript 파일 `node --check` 문법 검사 통과
- 데이터 검증 통과: 일반 8, 희귀 6, 영웅 4, 전설 2
- 고양이·스킬 ID 중복 없음, 등급별 스킬 규칙 통과
- 4/8지선다 개수·중복 없음·정답 포함·오름차순 통과
- 게임 상태 시작·정답·오답·종료 전이 통과
- 이전 저장 데이터의 스키마 2 마이그레이션 통과
- 모든 로컬 JavaScript 참조 파일 존재 확인

브라우저에서 실제 Firebase 인증·쓰기·읽기와 외부 CDN 네트워크는 자동 테스트하지 않았습니다. 배포 전 수동 확인이 필요합니다.

## 13. 확인된 문제

- 사용자 비밀번호와 관리자 자격 정보가 클라이언트 중심 구조이며 보안 개선이 필요합니다.
- 기존 보상은 20종 기본 카탈로그와 아직 직접 연결되지 않습니다.
- 실제 이미지·음원이 없습니다.
- 기존 관리자 화면은 요청된 콘텐츠 운영 설정 화면과 목적이 다릅니다.

## 14. 다음 단계 작업

20종 이미지/음원을 추가하고, 기존 보상 ID를 기본 카탈로그로 점진 매핑해야 합니다. 이후 비활성 모드를 하나씩 구현하고 주간 랭킹 데이터 정책, 콘텐츠 관리자 인증·설정 방식을 별도로 설계해야 합니다.

## 15. 수동 에셋

전체 목록과 파일명은 `ASSET_REQUIREMENTS.md`에 기록했습니다. 기본 고양이 이미지 20개, 정답 3개, 오답 2개, 영웅·전설·보상 음원이 필요합니다.

## 실행 방법

빌드 명령은 없습니다. 프로젝트 루트의 `index.html`을 정적 웹 서버 또는 GitHub Pages에서 열면 됩니다. 개발자 콘솔에서 `runPhase1Validation()`을 실행하면 기반 데이터 검증 결과를 확인할 수 있습니다.
