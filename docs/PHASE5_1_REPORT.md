# Phase 5.1 도감 정렬 보정 보고서

## 적용 결과

- 도감은 저장 데이터의 `ownedCatIds`에 포함된 고양이만 렌더링한다.
- `legacy`, `visible: false`, `previewOnly`, `developmentOnly` 고양이는 DOM을 만들지 않는다.
- 공통 정렬 서비스가 활성 시즌 → 최근 90일 이내 종료 시즌 → 기본 → 오래된 시즌 → 시즌 정보 누락 순서를 적용한다.
- 각 그룹 안에서는 전설 → 영웅 → 희귀 → 일반, `displayOrder`, 최근 획득일, 이름순으로 정렬한다.
- 시즌 정보가 누락된 보유 고양이는 삭제하지 않고 마지막 그룹에 표시하며 콘솔 경고를 남긴다.
- 도감에는 비어 있는 그룹이나 등급 제목, 잠금 카드, 빈 카드 자리를 만들지 않는다.
- 시즌 화면의 고양이 목록도 같은 보유 필터와 정렬 서비스를 사용한다.
- 사용자는 최신 시즌·등급순, 등급 우선, 최근 획득순, 이름순을 선택할 수 있고 설정은 저장된다.

## 검증

`validateCollectionSortOrder`, `validateSeasonSortMetadata`, `validateOwnedCatFiltering`을 서비스, 검증기와 `NyankoDebug`에 공개했다. Phase 5.1 자동 테스트는 미보유/legacy 제외, 활성·최근·기본·오래된·누락 시즌 순서, 등급 순서와 정렬 안정성을 검사한다.

## 최종 UI 정책 반영

- 시작 화면에는 56~88px 대표 고양이 프로필, 이름·등급·바꾸기 버튼과 보유 수 요약만 표시한다.
- 전체 카드는 별도 도감 화면에서만 렌더링한다.
- 도감에 카드/목록 보기, 이름 검색, 등급 및 현재·지난 시즌·기본 필터를 추가했다.
- 카드와 목록 보기는 동일한 필터·정렬 결과를 사용하며 선택 상태도 저장한다.
- 게임 문제·정답 버튼·피드백에는 게임용 둥근 글꼴 스택을, 긴 설명에는 일반 고딕 스택을 사용한다.
- 정답은 약 220ms 확대·복귀, 오답은 약 220ms 좌우 흔들림을 적용하고 모션 감소 설정을 따른다.
- `CAT_IMAGE_PRODUCTION_LIST.md`와 `CAT_IMAGE_PATH_VALIDATION.md`에 26종 원본과 공통 fallback 상태를 기록했다.
