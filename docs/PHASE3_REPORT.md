# Phase 3 완료 보고서

## 1. 작업 개요

게임 모드를 기본 게임, 타임어택, 구구단 대모험으로 재구성했습니다. 세로형 월드맵, 실제 플레이 가능한 3개 지역과 30개 스테이지, 별·잠금·보스·진행 저장을 구현하고 UI를 현대적인 모바일 캐주얼 게임 스타일로 개선했습니다.

## 2. 분석 결과

세부 내용은 `PHASE3_ANALYSIS.md`를 참고하십시오. 기존 공통 문제 생성기와 플레이 화면을 재사용하고 모험 전용 데이터·저장·흐름만 분리하는 방식을 선택했습니다.

## 3. 생성한 파일

- `js/data/worlds.js`
- `js/services/adventureService.js`
- `js/game/adventureEngine.js`
- `js/tests/phase3Validation.test.js`
- `docs/PHASE3_ANALYSIS.md`, `docs/PHASE3_REPORT.md`
- `backup/phase3-original/`

## 4. 수정한 파일

`index.html`, `css/style.css`, 게임 설정, 스테이지 데이터, 저장·랭킹 서비스, 검증, 도감 렌더링, 타임어택 엔진과 회귀 테스트를 수정했습니다.

## 5. 제거한 기능

사용자 화면, 모드 설정, 플레이·결과 엔진, 순위 허용 모드, 관리자 설정과 디버그에서 제거했습니다. 해당 Firebase 조회와 제출은 더 이상 실행하지 않습니다.

## 6. 보존한 기존 데이터

스키마 3에 있던 제거 모드 개인 최고 기록은 `legacy.endlessPersonalBest`에 보존합니다. 기존 Firebase 문서도 삭제하지 않습니다.

## 7. 최종 모드

`classic`, `timeAttack`, `adventure`만 활성화됩니다. 모험은 Firebase 순위에 제출하지 않습니다.

## 8. 디자인·글꼴·버튼

Jua 외부 폰트를 제거하고 Pretendard, Noto Sans KR, Apple SD Gothic Neo, Malgun Gothic 순의 고딕 스택을 적용했습니다. CSS 토큰으로 표면·본문·포인트·상태 색상을 통일했습니다. 버튼은 Primary, Secondary, Ghost, Danger 계층과 최소 44px 터치 영역을 사용합니다.

## 9. 월드·스테이지

지역 1 고양이 초원(2·3단), 지역 2 생선 항구(4·5단), 지역 3 장난감 숲(6·7단)에 각각 10개 스테이지가 있습니다. 지역 4~8은 준비 중 카드로 표시됩니다. 스테이지 유형은 일반, 시간, 콤보, 중간 보스, 최종 보스입니다.

## 10. 별·보스·해제

통과 시 1별, 유형별 2별 조건, 오답 없이 통과 시 3별입니다. 최고 별은 낮아지지 않습니다. 스테이지는 순차 해제되며 각 지역 10번 보스 완료 시 다음 구현 지역이 열립니다. 보스는 정답마다 체력이 1 감소하고 오답은 모험 목숨을 1 줄입니다.

## 11. 저장 마이그레이션

스키마 4에 `adventureProgress`를 추가했습니다. 해제 지역·스테이지, 클리어 목록, 최고 기록, 총 별, 최근 선택을 저장합니다. 기존 닉네임, playerId, 기본·타임어택 기록, 고양이, 음량 설정은 유지됩니다.

## 12. 도감

`collection === "base"`인 20종만 렌더링합니다. 등급 구성은 일반 8, 희귀 6, 영웅 4, 전설 2입니다. 기존 무작위 보상은 삭제하지 않고 `legacyRewards`로 남으며 현재 도감에서는 표시하지 않습니다.

## 13. Firebase

기본 게임 `users` 전체 순위와 타임어택 `modeRankings` 주간·전체 쿼리를 유지했습니다. 모험은 로컬 진행만 저장합니다. 기존 설정, 규칙, 컬렉션과 문서를 변경·삭제하지 않았습니다.

## 14. 모바일·접근성

320~768px 대응 그리드, safe-area 하단 여백, 44px 이상 조작 영역, 내부 스크롤 결과 카드, 잠금 `aria-disabled`, 별 텍스트 레이블, 보스 progressbar 속성, 화면 제목 포커스와 `prefers-reduced-motion`을 적용했습니다.

## 15. 테스트 결과

- 활성 모드 3종 검증 통과
- 월드 8개, 활성 지역 3개, 스테이지 30개 데이터 검증 통과
- 최초 1-1만 해제, 실패 시 잠금 유지, 성공 시 1-2 해제 통과
- 3별 저장 및 낮은 기록 방지 통과
- 기본 고양이 20종 검증 통과
- 스키마 3→4 기존 기록·설정·레거시 보존 통과
- 타임어택 랭킹 허용 모드 검증 통과

## 16. 발견한 오류와 남은 작업

실제 Firebase 규칙·인덱스와 외부 네트워크는 배포 환경 확인이 필요합니다. 고양이·월드·보스 전용 이미지는 아직 없으며 CSS 카드와 플레이스홀더로 표시됩니다. 지역 4~8, 고양이 카드 선택과 스킬 실행은 다음 단계 범위입니다.

## 17. Phase 4 준비

월드와 스테이지가 데이터 기반이므로 지역 4~8을 추가할 수 있습니다. 모험 진행과 선택 고양이 ID가 분리되어 있어 카드 선택·스킬 시스템 연결이 가능합니다.
