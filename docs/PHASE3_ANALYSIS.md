# Phase 3 사전 분석

## 기존 구조

정적 `index.html`과 전역 IIFE JavaScript 모듈 구조입니다. 클래식은 `app.js`, 타임어택과 제거 대상 모드는 `modeEngine.js`, Firebase 신규 순위는 `modeRankings`, 기존 순위는 `users`를 사용했습니다. 저장 스키마는 3, 도감 화면은 Firebase의 무작위 레거시 보상 배열을 직접 표시했습니다.

## 제거 대상

- `gameConfig.modes.endless`와 전용 목숨·난이도 규칙
- `modeEngine.js`의 시작·정오답·결과·디버그 분기
- 신규 순위 서비스의 해당 모드 허용과 조회
- 관리자 설정의 전용 목숨 값
- 화면과 CSS의 전용 카드

기존 로컬 최고 기록과 Firebase 문서는 삭제하지 않습니다. 스키마 4의 `legacy.endlessPersonalBest`에 로컬 최고 기록을 보존하고 현재 게임과 순위에서는 읽지 않습니다.

## 유지 대상

클래식 문제·점수·도감 계정 흐름, 타임어택 실제 시각 타이머, `users` 전체 순위, `modeRankings` 타임어택 주간·전체 순위, 기본 고양이 20종, 사운드와 이미지 오류 처리를 유지합니다.

## 모험에 재사용하는 기능

`questionGenerator.createOptions`, 정오답 사운드, 공통 `play-screen`, 입력 잠금 패턴, 저장 서비스와 CSS 이미지 대체 구조를 재사용합니다.

## 저장 마이그레이션

스키마 3의 프로필, 기본·타임어택 기록, 수집, 설정을 스키마 4에 병합합니다. 제거 모드 최고 기록은 `legacy`에 보존하고 과거 게임 기록은 현재 기록 목록에서 제외합니다. `adventureProgress`는 지역 1과 1-1만 열린 기본값으로 추가합니다.

## Firebase 호환

기존 컬렉션과 문서는 삭제하지 않습니다. 모험 결과는 Firebase에 제출하지 않고 타임어택만 신규 랭킹 서비스에 제출합니다. 기존 제거 모드 쿼리는 실행하지 않습니다.

## 디자인 문제와 계획

기존 Jua 글꼴, 원색 버튼, 두꺼운 입체 그림자, 과한 둥근 모서리와 보스 흔들림이 화면을 유아적으로 보이게 했습니다. 고딕 계열 폰트 스택, 제한된 보라·중성 팔레트, 얇은 테두리, 일관된 버튼 계층, 모바일 세로형 월드 카드로 변경합니다.
