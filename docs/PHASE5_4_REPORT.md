# Phase 5.4 완료 보고서

## 작업 개요

관리자 재화 Grid, 공통 답 버튼 반응, 카드팩 확률, 고양이 집 연출과 스킵/복구, 고양이 설명·연출 스킬, 중복 별, 미션 표시, 1-10 보스 진행 오류를 수정했다. 기존 사용자 ID와 고양이 ID 및 저장 재화는 초기화하지 않았다.

## 관리자 재화

- 원인: 가변 길이 사용자 ID와 버튼이 flex에 함께 있어 열 기준이 없었다.
- 구조: 데스크톱 4열(이름/내부 ID/수량/지급), 640px 이하 2열 재배치다.
- 코인·기본·고급·시즌 뽑기권을 Firestore `pendingResources`로 지급하고 다음 로그인에 자동 수령한다.
- 시즌 재화는 관리자에게만 보이며 일반 사용자 시즌 UI는 기능 플래그 false 상태를 유지한다.

## 게임 피드백

- 모든 답 버튼에 `game-answer-button`을 자동 부여해 hover, press, focus, 정답·오답 터치 영역을 통일했다.
- 모험에도 공통 정답·오답 사운드를 연결했다.
- 대표 고양이의 `presentationSkill.effectThemeId`가 공통 효과 레이어 테마가 된다.
- 성능형 영웅 스킬은 `skillActive=false`로 막아 점수·시간에 영향을 주지 않는다.

## 뽑기

- 코인/기본: 60/30/9/1, 고급/시즌: 45/35/17/3.
- 기본 팩 후보는 배포 가능한 기본 고양이만 사용한다. 시즌 후보는 분리되어 있고 플래그가 꺼진 동안 DOM에 생성하지 않는다.
- 로컬 SVG 고양이 집이 흔들리고 빛·문 열림 뒤 카드가 등장한다. 등급별 광채와 동작 줄이기를 적용했다.
- 결과 저장 후 연출하며 스킵은 같은 결과만 공개한다. 상태 잠금으로 연타를 차단한다.
- `pendingDrawReveal`로 새로고침 후 결과를 복구하고 확인 시 제거한다.
- 코인 부족 안내는 버튼 아래 별도 줄이다.

## 고양이 정보와 중복 별

- 표시 이름의 영웅/전설 단어를 제거하고 ID는 유지했다.
- 기본·시즌 26종 모두 데이터 설명을 추가했다.
- 영웅은 효과/사운드 테마, 전설은 추가 특별 옵션 하나를 표시한다.
- 중복 5장마다 단계가 오르며 금색 1~5개 후 빨간색 1~5개를 표시한다. 기존 조각 지급은 유지된다.
- 도감 썸네일·상세·뽑기 결과에 로컬 SVG 별과 접근성 레이블을 사용한다.

## 오늘의 미션

보스 미션 하나만 고급 뽑기권을 지급한다. 기존 중복 수령 방지는 유지했고 제목·보상·버튼이 모바일에서 줄바꿈되도록 했다.

## 스토리 1-10

- 정확한 원인: `boss-hp.textContent`가 자식 `boss-hp-bar`를 삭제했고 다음 줄의 null 접근 예외가 첫 답 처리와 다음 문제 예약을 중단했다.
- 수정: HP 텍스트와 막대 갱신 대상을 분리하고 null 안전 처리를 적용했다. HUD·효과·사운드 오류는 핵심 상태 전환을 막지 않는다.
- 상태: `stageIntro → questionActive → answerFeedback → questionActive`이며 700ms 정상 전환과 1800ms 안전 전환이 있다.
- 종료: 1-10은 12문제/HP 12, 정답당 HP 1 감소이며 HP 0, 문제 소진 또는 목숨 0에서만 결과로 이동한다.
- 회귀: 1-1, 1-5, 1-10, 2-5, 2-10, 3-5, 3-10에서 정답 3회 후 네 번째 문제, 잠금 해제, 보스 HP 감소를 자동 테스트해 모두 통과했다.

## 검증 결과

- 수정 JavaScript 문법 검사 통과.
- 4개 팩 확률 합 100% 및 10,000회 모의 분포 통과.
- 고양이 26종 설명/이름/연출 정책 통과.
- 고급권 미션 정확히 1개 통과.
- 로컬 파일 URL은 브라우저 자동화 보안 정책으로 화면 클릭 검증이 차단되어 DOM 회귀 테스트로 대체했다. 실제 기기 시각 확인은 남아 있다.
