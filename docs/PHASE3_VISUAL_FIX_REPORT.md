# Phase 3 시각 보정 완료 보고서

확인 날짜: 2026-07-22

## 1. 이전 카드 양식 확인 위치

이전 양식은 `backup/phase1-original/index.html`, `backup/phase2-original/js/app.js`, `backup/phase2-original/css/style.css`에서 확인했습니다. 등급 배지, 이미지 중심 프레임, 등급별 배경·테두리, 카드 단위 클릭 구조를 사용하던 형태였습니다. Git 저장소에는 비교할 수 있는 커밋 이력이 없어 백업 파일을 기준으로 복구했습니다.

## 2. 복구한 카드 요소

- 기존 등급 배지와 등급별 테두리·배경
- 세로형 이미지 중심 카드 비율
- 획득/미획득 상태와 실루엣
- 이름, 선택 안내, 대표 고양이 표시
- 카드 클릭을 통한 대표 고양이 설정
- 전체·일반·희귀·영웅·전설 필터

## 3. 빈 슬롯 원인과 제거

Phase 3 카드가 고정 폭의 작은 요소로 표시되어 화면 폭에 따라 비어 보이는 열이 생겼습니다. 렌더링 전에 `collection === "base"`, `visible !== false`, 활성 시즌 조건으로 데이터를 먼저 필터링하고, 실제 표시 데이터만 DOM에 생성하도록 변경했습니다. legacy 데이터는 DOM에 생성하지 않습니다. `auto-fit/minmax` 기반 그리드를 사용해 필터 전환 후에도 빈 슬롯이 남지 않습니다.

## 4. 고양이 데이터 확인

기본 도감은 일반 8, 희귀 6, 영웅 4, 전설 2로 총 20마리입니다. 미획득 기본 고양이는 동일한 카드 프레임과 어두운 실루엣으로 표시됩니다. 원본 WebP가 아직 없는 경우 내부 `cat-placeholder.svg`로 교체되어 깨진 이미지가 나오지 않습니다.

## 5. 생성한 월드·보스 에셋

- 월드: 고양이 초원, 생선 항구, 장난감 숲, 잠금 지역 SVG
- 보스: 월드별 중간 보스와 최종 보스 총 6종 SVG
- 장식: 털실, 생선 상자, 장난감 기어, 스테이지 자물쇠 SVG
- 대체 이미지: 모험 배경, 보스, 고양이 SVG

모든 경로는 GitHub Pages에서 동작하는 프로젝트 상대 경로입니다. 외부 이미지와 CDN은 사용하지 않았습니다.

## 6. 이미지 데이터와 fallback

월드 데이터의 `artwork`, `decoration`, `midBoss.image`, `finalBoss.image`에 경로를 저장했습니다. 스테이지 생성 시 `boss.image`와 `bossType`을 전달합니다. 로딩 실패 시 공통 SVG로 한 번 교체하고 `console.warn("[Adventure asset missing]", imagePath)`를 출력하며 게임은 계속됩니다.

## 7. 화면 보정

월드 카드는 배경 이미지, 어두운 그라데이션, 지역 번호·이름·출제 단·진행·별·버튼을 함께 표시합니다. 스테이지 화면은 선택 월드 배경과 반투명 표면, 지역 장식을 사용합니다. 보스전은 보스 이미지, 체력 바, 선택 고양이, 220ms 피격 이동과 데미지 숫자를 표시합니다. 모션 감소 설정에서는 피격 애니메이션을 제거합니다.
