# Phase 5.5.1 스토리 반영 보고서

## 데이터와 ID

- 월드: `js/data/worlds.js`
- 스테이지: `js/data/stages.js`
- 실제 ID: `world_01`, `stage_01_01`
- 병합: `adventureStoryData.js`가 월드 번호와 스테이지 번호를 실제 ID로 변환해 스토리 필드만 추가한다.

## 반영 범위

프롤로그 1개, 월드 1~3 인트로, 일반 스테이지 포함 30개 인트로, 1/2/3월드의 5·10 보스 인트로, 각 10단계 clearStory를 실제 실행 흐름에 연결했다.

## 저장

계정별 save의 `adventureStory` 아래 `prologueSeen`, `seenWorldIntros`, `seenStageIntros`, `seenClearStories`를 추가한다. 기존 필드가 없는 사용자는 접근 시 생성되며 기존 진행·보상·재화는 유지된다.

## UI와 접근성

동화풍 dialog, 제목 연결, 실제 텍스트 대사, 포커스 이동, Escape 닫기, 동작 줄이기를 지원한다. 420px 이하에서 이미지와 패널을 축소하고 내부 스크롤을 허용한다.

## 검증

- 월드 인트로 3/3
- 스테이지 인트로 30/30
- 보스 인트로 6/6
- clearStory 3/3
- 병합 전후 문제 수·보스 HP·스테이지 유형 동일
- 1-1, 1-5, 1-10, 2-5, 2-10, 3-5, 3-10 세 문제 연속 진행 통과
- 각 최종 보스는 세 정답 후 HP 12→9, 입력 잠금 해제, 네 번째 문제 생성

## 남은 문제

로컬 file URL 브라우저 자동화가 보안 정책상 차단되어 320~1024px 실제 렌더링은 수동 시각 확인이 필요하다. 코드와 CSS 반응형 규칙 및 DOM 회귀 테스트는 통과했다.
