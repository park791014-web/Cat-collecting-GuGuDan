# Phase 5.1 UI 완료 보고서

## 시작 화면

- 대표 고양이는 56~88px 프로필 이미지, 이름, 등급, 바꾸기 버튼만 표시한다.
- 보유 고양이 수와 기본 컬렉션 진행률을 간결하게 표시한다.
- 전체 도감 카드는 별도 도감 화면에서만 생성한다.

## 도감 표시 정책

- `ownedCatIds`에 존재하며 실제 데이터가 있는 고양이만 대상으로 한다.
- legacy, `visible:false`, `previewOnly`, `developmentOnly`는 DOM을 만들지 않는다.
- 미보유 잠금 카드, 빈 슬롯, 투명 자리표시자는 사용하지 않는다.

## 탐색과 정렬

- 목록 보기와 카드 보기를 지원하고 기본값은 목록이다.
- 부분 이름 검색, 등급 필터, 현재·최근·기본·지난 시즌 필터를 제공한다.
- 기본 정렬은 활성 시즌 → 최근 90일 종료 시즌 → 기본 → 오래된 시즌 → 정보 누락 순서다.
- 그룹 내부는 전설 → 영웅 → 희귀 → 일반, `displayOrder`, 최근 획득일, 한국어 이름순이다.
- `collectionSortService.js`를 도감과 시즌 보유 목록이 함께 사용한다.

## 글꼴과 피드백

- 긴 UI 문장은 고딕 계열, 문제·정답 버튼·게임 피드백은 둥근 게임용 글꼴 스택을 사용한다.
- 정답은 약 220ms 확대·복귀, 오답은 약 220ms 좌우 이동을 사용한다.
- 운영체제 모션 감소와 게임 설정의 모션 감소를 모두 반영한다.

## 이미지

- 기본 20종, 시즌 1 6종, 공통 fallback 1종으로 총 27개 항목을 문서화했다.
- 데이터 경로·중복·수량·fallback 연결을 `validateCatImageManifest`로 검증한다.
- 원본 WebP는 아직 제작 대기 상태이며 누락 시 `[Cat image missing]` 경고 후 공통 SVG fallback을 표시한다.

## 접근성과 모바일

- 검색 label, 보기·필터의 `aria-pressed`, 대표 고양이의 `aria-selected`, 모달 초점 복귀를 적용했다.
- 조작 영역은 최소 44px이며 320px에서 고정 2열이나 가로 스크롤을 강제하지 않는다.
- 하단 safe area를 고려한다.

## 생성·수정 파일

- 생성: `collectionSortService.js`, `phase51Validators.js`, Phase 5.1 테스트, UI 분석·보고서, 이미지 제작·검증 문서.
- 수정: `index.html`, `style.css`, 저장 서비스, 고양이 데이터, 도감·시즌 컨트롤러, 이미지 fallback 처리.
