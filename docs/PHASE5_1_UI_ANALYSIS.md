# Phase 5.1 UI 사전 분석

## 기존 구조

- 시작 화면의 `collection-box`가 요약과 전체 도감 역할을 동시에 담당해 게임 모드보다 큰 공간을 차지했다.
- 도감 렌더링은 `collectionController.js`, 보유 데이터는 스키마 6의 `collection.ownedCatIds`와 `catProgress`에 있다.
- 기본·시즌 고양이는 별도 데이터 파일이며 시즌 상태는 `seasonService.getSeasonStatus`가 한국 시간 오프셋을 기준으로 판정한다.
- 상세 모달과 장착 로직은 재사용 가능하며 카드팩 서비스가 신규 소유권과 `obtainedAt`을 저장한다.

## 발견한 위험

- 시작 화면과 도감이 같은 DOM ID를 공유하면 숨겨진 카드가 시작 화면에 영향을 줄 수 있다.
- 필터 뒤 CSS로 카드를 숨기면 미보유·비공개 고양이가 DOM에 남는다.
- 기존 저장값 `card`, `obtained`는 최종 허용값 `grid`, `recently_obtained`와 다르다.
- 실제 WebP 원본 26개가 아직 없으므로 공통 fallback과 경고가 필수다.
- 기존 고양이 데이터의 한글 표시 이름 일부가 손상되어 최종 정책에 기재된 실제 ID·이름으로 복구할 필요가 있었다.

## 적용 전략

- 시작 화면은 대표 고양이 프로필과 보유 통계만 렌더링하고 별도 도감 화면을 사용한다.
- 보유·공개 필터 → 검색 → 등급 → 컬렉션 → 공통 정렬 → 그룹 → DOM 순서를 고정한다.
- 목록과 그리드는 동일한 정렬 배열을 사용한다.
- 설정값은 스키마를 올리지 않고 기존 설정 객체에 기본값을 병합하며 잘못된 값만 안전하게 복구한다.
- Firebase 설정, 랭킹, 게임 모드와 저장 ID는 변경하지 않는다.
