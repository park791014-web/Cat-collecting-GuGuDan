# Phase 5 분석

## 기존 구조

- 정적 HTML/CSS/JavaScript 웹 게임이며 Firebase 8.x를 로그인과 랭킹에 사용한다.
- 로컬 진행 데이터는 `gugudanV2Save`에 저장되며 Phase 4 스키마를 유지한 채 확장할 수 있다.
- 별도 서비스 워커나 캐시 저장소는 없다. 따라서 서비스 워커를 새로 도입하지 않고 버전 쿼리와 중앙 버전 상수로 갱신을 식별한다.

## 구현 방침

- 시즌 정의, 시즌 고양이, 시즌 미션을 각각 데이터 파일로 분리한다.
- 날짜 판정은 ISO 8601의 `+09:00` 오프셋을 사용해 한국 시간 경계를 명시한다.
- 시즌 종료 후 신규 팩 개봉은 막되 이미 획득한 고양이는 도감과 장착에서 유지한다.
- 시즌 보상은 수동 수령이며 입장 보상·미션 보상·팩 개봉은 저장 데이터의 중복 상태로 보호한다.
- Firebase 설정과 보안 규칙, 기존 랭킹 컬렉션은 변경하지 않는다.

## 저장 마이그레이션

스키마 6은 시즌 티켓, 시즌 진행, 온보딩, 접근성 설정, 앱/콘텐츠 버전을 추가한다. 기존 프로필, 기록, 모험 진행, 보유 고양이와 재화는 병합하여 보존한다.
